RestPDF

A Self-Hostable, Open Source PDF REST API

RestPDF provides a simple and powerful RESTful API for your PDF manipulation needs. It's designed to be easily deployed on your own infrastructure, giving you full control over your documents and workflows.

How to Deploy RestPDF

Prerequisites

  • Java Development Kit (JDK): Version 8 or higher.
  • sbt (Simple Build Tool): The standard build tool for Scala projects.

Step 1: Clone the Repository

Get the source code from GitHub.

git clone https://github.com/Product-BU/restpdf.git
cd restpdf

Step 2: Build the Application

Use `sbt` to compile and package the application.

sbt dist

This creates a distributable `.zip` file in the `target/universal/` directory.

Step 3: Unzip the Package

Unzip the file created in the previous step.

unzip target/universal/restpdf-1.0.zip

Step 4: Run the Application

Navigate into the new directory and use the script in `bin/` to start the service.

bin/restpdf -Dplay.http.secret.key='your-super-secret-key' -Dconfig.file=conf/prod.conf -Dpidfile.path=/dev/null

Important Run-Command Flags

  • -Dplay.http.secret.key: **You must replace** `'your-super-secret-key'` with a long, random, and unique string for session security.
  • -Dconfig.file: Points to the production configuration file. You may need to create a `prod.conf` or modify `common.conf`.
  • -Dpidfile.path: Specifies the process ID file path. Using `/dev/null` is common to prevent its creation.
